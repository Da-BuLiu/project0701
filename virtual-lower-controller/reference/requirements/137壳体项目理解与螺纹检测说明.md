# 137 壳体自动检测项目：项目理解与螺纹检测说明

> 阅读依据：`ER-PM-001 技术协议-137壳体_0703.docx`、`软件需求规格说明书.docx`。本文将文档中**已明确的要求**与需要在项目启动前关闭的**待确认项**分开说明；未在原始文件中出现的螺纹规格、测点和公差，不作臆测。

## 1. 一句话理解项目

这是一个面向 137 壳体的**自动化尺寸检测与质量追溯系统**：工件由机器人上料，经多种视觉/传感器工位完成 52 项尺寸或特征检测，系统按公差输出 OK/NG，将合格件与不合格件分流，并以 SN 为主线保存原始数据、结果、图片和报告。

项目要替代游标卡尺、千分尺、投影仪、三坐标、三针、螺纹规、塞规等人工检测方式，重点解决人工影响大、判定不一致、数据难追溯的问题。

典型业务流如下：

```text
人工转运至上料台
  → 机器人 + 3D 引导相机取件/定位
  → 多工位采集（2D 相机、线光谱、内窥镜、螺纹设备）
  → 算法计算 52 项结果 + 公差判定
  → 上位机显示、存库、生成报告
  → PLC/机器人将工件送至 OK 料框或 NG 待检区
```

## 2. 壳体的螺纹检测，到底检测什么？

这里其实有两类相关检测，不能混为一谈：

| 类别 | 检测对象/内容 | 使用方式 | 本项目上位机职责 |
|---|---|---|---|
| 内窥镜尺寸检测 | 螺纹孔及内腔的**内径、孔深、台阶高度**等几何尺寸 | 100 万像素内窥镜伸入内腔，非接触式采集并计算 | 接入图像/测量结果，显示、判定、存储；算法和接口细节仍需确认 |
| 独立螺纹检测系统 | “其余类型螺纹”的**螺纹精度、通止规性能**等关键指标 | 独立的、客户提供的螺纹检测模块 | 向模块提供产品 SN，读取其回传数据，在主界面两个小框显示并归档；**不干预其检测流程和算法** |

### 2.1 应如何理解“通止规性能”

它是对螺纹装配/功能性的检验：通规应能按要求旋入（或通过），止规不应越过允许位置。它不是单纯量一个孔径，而是综合反映螺纹是否可正常配合。文档还提到“螺纹精度”，通常可能关联牙型、螺距、中径、有效旋合等能力；但**本项目文件没有给出具体螺纹型号、检测项、测量值定义或判定阈值**，不能据此直接开发算法或验收。

### 2.2 当前可确定的边界

- 螺纹检测模块标注为“客户提供”。
- 上位机需要以当前工件 SN 将螺纹结果与其它检测结果关联。
- 不采用弹窗；主界面底部放置两个结果小框。
- 螺纹未检出或未加工时，界面以黄色提示；正常结果至少应有 OK/NG 状态，数值显示至少保留 3 位小数（若该结果为数值型）。
- 上位机不是螺纹机的控制器，不负责启动其检测、运动控制或核心检测算法。

### 2.3 螺纹部分必须补齐的需求

在联调前，客户/螺纹设备供应方必须提供：

1. 螺纹清单：位置、内/外螺纹、规格、左右旋、图纸公差、检测项目与产品级判定规则。
2. 结果字典：两个小框分别代表什么；每项的值、单位、OK/NG/未检、错误码和时间戳定义。
3. 交互协议：SN 字段、触发时机、通信方式/IP/端口、报文示例、应答、超时、重试、断线恢复及一件工件的完成标志。
4. 数据归档：原始结果、图片/曲线（如有）、设备版本、人工复核值分别是否保存，以及保存字段与路径。
5. 异常与验收：无螺纹、漏检、重复 SN、结果迟到、设备离线时的处置；螺纹项目是否纳入最终 OK/NG、复检和 600 件验收样本的规则。

## 3. 系统包含的模块及功能

### 3.1 现场设备/检测模块

| 模块 | 主要功能 | 主要数据或设备 |
|---|---|---|
| 上料、下料机器人 | 工件取放、OK/NG 分拣 | 2 路 3D 引导相机生成点云，计算偏移/抓取参数；机器人动作由 PLC/机器人执行 |
| 端面检测 | 左右端面尺寸、SN 识别 | 2 × 2000 万像素相机及光源 |
| 总长测量 | 壳体总长等尺寸 | 1 × 2000 万像素相机及光源 |
| 端面与柱面检测 | 左右负端面、柱面两孔尺寸；按左端面槽进行柱面特征姿态引导 | 2 × 2000 万 + 1 × 630 万像素相机及光源 |
| 同轴线光谱 | 扫描柱面/点云，计算孔特征；场景描述还包括内径、角度、台阶深度 | 4020 型同轴线光谱传感器 |
| 内窥镜检测 | 内腔及螺纹孔相关尺寸 | 100 万像素内窥镜 |
| 独立螺纹检测 | 螺纹精度、通止规性能等 | 客户提供；向上位机回传数据 |
| PLC 与安全联动 | 工位节拍、设备状态、报警、动作联锁 | PLC 是设备动作主体，上位机发命令并同步/记录状态 |

### 3.2 上位机软件模块

| 模块 | 具体职责 |
|---|---|
| 设备主监控/22 寸小屏 | 展示 8 个工位、当前 SN、采集图像、52 项结果、OK/NG 和螺纹两个小框；生产时外触发刷新，调试可实时预览；提供启动、停止、暂停、复位、回原点等入口 |
| 100 寸大屏 | 展示车间生产概览、当前检测结果，可接海康监控相机；规划为“项目/生产宏观”和“当前结果”两页，可考虑 3D 模型映射 |
| 检测编排与结果判定 | 接收各模块结果，统一单位/状态，按基准与公差生成项目 OK/NG 和产品汇总结论；支持未检出项人工补录 |
| 外设接入适配层 | 统一接入相机、点云、内窥镜、PLC、机器人、螺纹模块；真实 SDK 未具备时可用仿真适配器联调 |
| 数据管理与追溯 | 以 SN 为唯一标识，保存检测记录、图片、结果、人工修改及审计痕迹；按 SN 查询、导出 Excel、生成日报/月报/年报和检测报告 |
| 用户、权限、班次 | 操作员仅基础控制和查看；工程师可配置、数据管理、维护；系统管理员管理账号/重置密码；记录登录日志和班次 |
| 维护与调参 | 面向工程师/维护人员的设备连接、算法版本、维护操作、数据库与诊断能力；操作员不可进入 |
| 统计与 MES/ERP 预留 | 班次累计检测数、OK 数、NG 数、良率等；为 MES/ERP 输出或上传预留接口 |

## 4. 软件设计需要做哪些工作

### 4.1 先完成需求与接口基线

- 建立《52 项检测项清单》：编号、工位、名称、特征类型、单位、基准/公差、算法版本、结果来源、判定规则、是否可人工补录。
- 为每类设备定义统一的采集任务、原始数据、计算结果、状态、错误码和完成事件模型；螺纹模块须单独形成 ICD（接口控制文档）。
- 与 PLC 明确状态机与联锁：启动条件、暂停/停止语义、复位/回原点、急停、报警确认、超时和失败恢复。

### 4.2 架构与核心实现

- 在 Windows 11 IoT 企业版工控机上部署独立桌面程序；本地 SQLite + 文件系统保存结构化结果、图片、点云和报告。
- 将“设备适配、采集/算法任务、业务流程与判定、数据持久化、UI”解耦，防止某个 SDK 或设备更换牵动整个系统。
- 实现按 SN 的工件会话（workpiece session）：产生 SN/读码 → 关联每站任务 → 接收异步结果（含螺纹结果）→ 汇总判定 → 归档 → 允许安全地迟到补写或复检。
- 对人工补录、修改、复检建立不可覆盖的审计记录：原值、新值、原因、操作者、时间、影响的最终结论。
- 实现外设断线重连、结果超时、重复/错配 SN、磁盘不足、SQLite 损坏/备份恢复等故障处理与日志。

### 4.3 界面、数据和交付

- 完成小屏、大屏、数据查询/导出、账户权限、检测配置、维护、报警、班次和统计页面；明确未检、未加工螺纹、设备离线、检测失败的视觉状态。
- 设计数据库表、索引、目录与命名规则；实现 90 天保留策略、容量预警、备份、自动清理/归档（策略须经确认）。
- 完成报表模板、Excel 导出、检测报告及 MES/ERP 可扩展接口。
- 编写接口文档、部署手册、操作手册、维护/调参手册；按要求交付核心边缘轮廓提取算法源代码和鲁棒性 Demo。

## 6. 关键指标与验收认识

- 52 个检测尺寸；静态重复性要求分为 0.1 mm（10 项）、0.05 mm（10 项）、0.03 mm（32 项）。
- 600 件良品/不良品均应能输出尺寸结果；过杀率不高于 3%，漏检率为 0%，读码成功率为 100%。
- CT 不高于 180 秒。技术协议按“上料机器人拍照至下料进入 OK/NG 框”计时，但另一份材料写了不含上下料时间，必须统一口径。
- 本地保存周期在两份材料中曾有“暂定 2 年”和“会议确认 90 天”两种说法；目前应以会议确认的 90 天作为暂行基线，同时在需求评审中正式签字。

## 7. 当前主要问题、风险和建议优先级

| 优先级 | 问题/风险 | 影响 | 建议动作 |
|---|---|---|---|
| P0 | 螺纹规格、测点、结果字典、接口和产品级判定未定义 | 无法完成螺纹接入、界面、存库与验收 | 召开三方（客户、螺纹机、上位机）接口会，签署 ICD 和验收用例 |
| P0 | 52 项的最终公差、顺序、来源算法与汇总判定规则未冻结 | 无法可靠实现 OK/NG，也不能做回归测试 | 发布受控的检测项主数据表，变更走版本与审批 |
| P0 | PLC/机器人通信、状态机、报警码、联锁与超时未确定 | 有设备安全和节拍风险 | 输出 PLC I/O/报文表与时序图，先用仿真器联调再接现场 |
| P0 | CT 起止边界存在矛盾 | 验收争议 | 以技术协议或双方签字的验收方案明确唯一计时点 |
| P1 | 图片/点云数量、体积、保留与清理策略不明确 | 磁盘耗尽或追溯不足 | 估算日数据量，定义原始数据分级、90 天策略、容量预警和备份恢复演练 |
| P1 | 人工补录、复检、NG 处置是否改变最终结果和统计未确定 | 数据可信度、良率报表会失真 | 定义原始结果、复检结果、放行结论三层数据及统计口径 |
| P1 | 标定、坐标系、重复性和 GR&R 方案尚未具体化 | 0.03 mm 级项目可能无法稳定达标 | 对每类传感器建立标定 SOP、标准件、环境条件和重复性试验计划 |
| P1 | 本地 SQLite 单机方案的备份、断电恢复、权限与防篡改未细化 | 追溯数据丢失或被非授权修改 | 增加自动备份、校验、账户权限、审计日志和恢复演练 |
| P2 | 大屏视觉、海康接入、MES/ERP 范围尚未冻结 | 影响后期工作量与排期 | 与核心检测解耦，采用接口预留和里程碑验收 |

## 8. 建议的近期推进顺序

1. 冻结产品图纸、52 项检测主数据、螺纹检测范围和最终 OK/NG 规则。
2. 完成 PLC/机器人/螺纹设备/相机的接口控制文档与仿真器。
3. 按“单工位可采集 → 单项可计算 → 全工件按 SN 汇总 → 数据可追溯 → 自动分拣”逐层联调。
4. 以标准件、良品、典型 NG 件执行重复性和误判测试，形成预验收报告。
5. 现场联调后按统一 CT、600 件样本和交付文档完成验收；培训至少覆盖操作、大屏/小屏、算法配置调参、数据库和软件维护。

## 结论

该项目的核心不是单一视觉测量软件，而是一个把多传感器、机器人、PLC、独立螺纹机和质量数据串成闭环的自动化检测系统。螺纹部分在系统中是一个**外部检测能力的集成点**：上位机负责 SN 关联、结果接收、显示、归档和参与最终质量闭环；独立螺纹设备负责实际检测与算法。项目能否按期稳定验收，最优先取决于螺纹接口、52 项公差/判定和 PLC 时序这三类边界能否尽快书面冻结。
