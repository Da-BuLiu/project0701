# 三个位置均值差异与最终长度选择分析

本文继续只回答问题和解释验收/统计口径，不优化代码。

## 1. 场景 B 和场景 C 到底有什么区别

之前写的两个场景是：

```text
场景 B：三个 pos 是同一个尺寸的三次换位拍摄
场景 C：三个 pos 是同一尺寸，但算法测到的不是同一物理边界
```

它们不是完全并列、互斥的两类数据，而是两个不同层面的描述。

| 层面 | 描述 | 回答的问题 |
|---|---|---|
| 场景 B | 数据采集设计 | 这批图本来想测什么 |
| 场景 C | 算法实际行为 | 算法实际上测到了什么 |

所以更准确的说法是：

```text
这批数据的测试意图属于场景 B；
但当前算法结果表现出场景 C 的风险。
```

也就是：你本来是把同一个零件放到 3 个不同位置拍摄，想验证同一个侧边长在不同位置是否还能测一致；但当前算法可能没有始终测到同一条真实物理侧边，而是跟着不同位置下的高亮反光区域走了。

## 2. 当前数据更像哪一种情况

从目录结构看：

```text
sample_1/pos_1
sample_1/pos_2
sample_1/pos_3

sample_2/pos_1
sample_2/pos_2
sample_2/pos_3
```

每个 `sample` 下有 3 个 `pos`，每个 `pos` 有 20 张图。这个结构很像：

```text
同一个零件，在 3 个不同拍摄位置或姿态下，各拍 20 张。
```

因此我认为采集设计上应该属于：

```text
场景 B：同一个尺寸的三次换位拍摄。
```

但从测量结果看，三个位置的均值不是小幅随机波动，而是明显有位置相关的系统偏差。

当前结果：

| sample | pos | mean_px | range_px | mean - reference_px |
|---|---:|---:|---:|---:|
| s1 | pos1 | 2737.126519 | 0.036946 | -0.000669 |
| s1 | pos2 | 2733.747456 | 0.077381 | -3.379732 |
| s1 | pos3 | 2716.188775 | 0.096568 | -20.938413 |
| s2 | pos1 | 2736.058237 | 0.441362 | -1.068951 |
| s2 | pos2 | 2734.229291 | 0.084814 | -2.897897 |
| s2 | pos3 | 2723.582433 | 0.101974 | -13.544755 |

这里的 `reference_px` 是当前标注参考长度：

```text
manual_length_px = 2737.127188 px
```

另外，`s1_label.bmp` 和 `sample_1/pos_1/Pic_2026_04_23_142727_286.bmp` 的文件哈希完全一致，说明当前标定图就是 `sample_1 pos_1` 的一张图。因此 `s1 pos1` 均值接近参考长度并不意外，它是被标定锚定住的位置。

关键现象是：

```text
s1: pos1 到 pos3 差约 20.94px
s2: pos1 到 pos3 差约 12.48px
```

这不是同一位置内的随机波动，而是换位置后均值整体移动。

所以当前状态应该表述为：

```text
组内静态重复性好；
位置间一致性差；
疑似算法测量目标随位置变化，也就是场景 C 风险。
```

## 3. 为什么不是简单说“场景 B 不合格”就完了

如果只说“场景 B 不合格”，容易漏掉原因。

场景 B 的验收目标是：

```text
同一个零件、同一个尺寸，换 3 个位置拍，结果仍然一致。
```

当前结果没有做到这一点。

但为什么没做到，有几种可能：

1. 相机/镜头在不同位置的像素当量不同，没有做位置标定或畸变校正。
2. 零件在不同位置的姿态、倾斜、透视不同，导致图像上的投影长度不同。
3. 算法测的是高亮反光条，不是真实物理端面。
4. 不同位置下高亮区域的左右边缘变化，算法稳定地测到了不同的反光边界。
5. 标定只基于 `pos1`，直接套到 `pos2/pos3`，导致跨位置系统偏差。

从当前叠加图和代码逻辑看，第 3、4、5 点风险比较明显。

也就是说，当前不是“随机误差大”，而是：

```text
算法在每个位置内部都很稳定，
但换位置后稳定地偏到另一个长度。
```

这种问题应该归类为：

```text
位置间一致性/再现性问题
系统误差问题
测量目标定义或定位算法问题
```

不是单纯的静态重复性问题。

## 4. 那三个 pos 的均值到底该选哪个

不能按“哪个组内极差小”来选。

原因很简单：

```text
极差小只说明稳定，不说明准确。
```

例如 `s1 pos1` 的 `range_px = 0.036946`，`s1 pos2` 的 `range_px = 0.077381`，`s1 pos3` 的 `range_px = 0.096568`。这三组都很稳定，但 `pos3` 的均值比参考短约 20.94px。

如果一个错误边界每次都被稳定地找到，它的极差也会很小。

所以不能这样选：

```text
选择组内极差最小的 pos 作为零件侧边长。
```

这会把“重复性好”误当成“测得准”。

## 5. 正确选择均值的原则

最终选哪个均值，必须先回答一个问题：

```text
哪个 pos 的测量结果最接近真实物理侧边长？
```

这个问题不能靠当前 JSON 自己回答，因为当前 JSON 只有算法测量值，没有真实标准值。

正确选择有三种情况。

### 5.1 有外部标准值时

这是最可靠的情况。

例如用卡尺、CMM、标准件、治具标定或人工精标得到真实长度：

```text
L_true_mm
```

然后每个位置都计算：

```text
pos_mean_mm = pos_mean_px * mm_per_px
bias_mm = pos_mean_mm - L_true_mm
repeatability_mm = pos_range_px * mm_per_px
```

选择规则应该是：

```text
先看 bias 是否小；
再看 repeatability 是否满足；
最后看稳定性和工程可维护性。
```

也就是：

```text
准确度优先，重复性其次。
```

如果某个位置极差很小但偏差很大，不能选。

### 5.2 没有外部标准值，但验收只固定一个拍摄位置

如果未来产线只会在一个固定位置测量，比如永远用 `pos1`，那么可以选择：

```text
固定测量位置的均值。
```

当前数据里，如果产线固定使用 `pos1`，那么 `pos2/pos3` 可以作为换位鲁棒性参考，但不参与最终长度输出。

但这必须在验收方案里写清楚：

```text
尺寸输出只基于固定工位/固定位置。
```

否则验收方会问：

```text
为什么 pos2/pos3 不算？
```

### 5.3 没有外部标准值，且三个位置都要求可用

如果验收要求同一个零件放在 3 个位置都能测同一个侧边长，那么现在不能随便选一个均值作为最终答案。

这时应该判定：

```text
当前跨位置一致性不满足，需要先解决位置间偏差。
```

此时不能用下面这些做法掩盖问题：

```text
选最小极差的 pos
选最大均值的 pos
选最接近标定值的 pos
三个 pos 简单平均
去掉最差的 pos
```

因为这些都是事后挑选，没有物理依据，也不能解释为什么被选中的值就是真实侧边长。

## 6. 为什么不能简单取三个 pos 的平均值

有人可能会想：

```text
既然三个位置都有结果，那取平均值不就行了吗？
```

这在当前数据下不建议。

原因是三个位置的差异不是随机噪声，而是系统偏差。系统偏差不能靠平均消除。

以 `s1` 为例：

```text
pos1 mean = 2737.126519 px
pos2 mean = 2733.747456 px
pos3 mean = 2716.188775 px
```

三者简单平均：

```text
(2737.126519 + 2733.747456 + 2716.188775) / 3 = 2729.020917 px
```

这个平均值既不是 `pos1` 的真实测量，也不是 `pos2` 或 `pos3` 的真实测量；如果 `pos3` 是算法测错边界，平均值还会被错误结果拉偏。

所以：

```text
随机误差可以平均；
系统误差不能直接平均。
```

## 7. 当前如果必须给一个临时判断

如果只基于当前已有证据，不能正式证明哪个就是“真实侧边长”。

但可以做一个工程判断：

```text
pos1 更像当前算法的标定位置；
pos2/pos3 表现出相对 pos1 的系统偏短；
因此当前算法输出若要临时参考，只能把 pos1 当作标定条件下的结果，而不能把 pos1/pos2/pos3 混为同一个可靠长度。
```

注意，这不是说 `pos1` 一定是真实长度。它只是说：

```text
当前标定是基于 pos1 建立的；
pos1 和参考标注一致；
所以 pos1 是当前算法链路里最有依据的结果。
```

正式验收时仍然需要外部真实长度验证。

## 8. 如果三个位置都是同一个侧边长，应该怎么判断

建议把判断拆成三步。

### 8.1 第一步：每个位置内部看静态重复性

对每个位置算：

```text
repeatability_px = max(length_px) - min(length_px)
repeatability_mm = repeatability_px * mm_per_px
```

当前 6 组的组内 `range_px` 都很小，说明这一项表现不错。

### 8.2 第二步：每个位置和真实标准比准确度

对每个位置算：

```text
bias_px = pos_mean_px - true_length_px
bias_mm = bias_px * mm_per_px
```

没有 `true_length_px` 或 `true_length_mm` 时，这一步做不了。

这一步是决定能不能选某个位置作为最终长度的关键。

### 8.3 第三步：三个位置之间看再现性

对同一个样品的三个位置算：

```text
position_range_px = max(pos_mean_px) - min(pos_mean_px)
position_range_mm = position_range_px * mm_per_px
```

当前结果：

```text
s1 position_range_px = 20.937744 px
s2 position_range_px = 12.475804 px
```

如果按之前临时估算的 `1px ≈ 0.0072mm`：

```text
s1 position_range_mm ≈ 0.150752mm
s2 position_range_mm ≈ 0.089826mm
```

这说明跨位置一致性明显弱于组内静态重复性。

## 9. 应该如何定义最终报表口径

如果客户验收的是“同一个零件换 3 个位置也要测同一个侧边长”，报表不应该只输出一个模糊的 `length`。

建议报表至少拆成：

```text
pos1_mean
pos1_repeatability
pos1_bias

pos2_mean
pos2_repeatability
pos2_bias

pos3_mean
pos3_repeatability
pos3_bias

position_mean_range
final_judgement
```

最终判定逻辑应该类似：

```text
每个 pos 的组内重复性都合格；
每个 pos 的准确度都合格；
三个 pos 的位置间均值差也合格；
才认为这个尺寸项在三位置条件下合格。
```

如果只要求固定位置测量，则报表应写成：

```text
final_length = pos1_mean
pos2/pos3 仅作为验证数据，不参与最终长度输出
```

这两种验收口径差别很大，必须提前定。

## 10. 当前最合理的结论

针对你的两个问题，结论如下。

### 10.1 B 和 C 的区别

`B` 是采集目的：

```text
同一个零件、同一个侧边长，放到 3 个不同位置拍摄。
```

`C` 是算法风险：

```text
虽然想测同一个侧边长，但算法在不同位置可能跟着反光区域走，实际测到的不是同一条物理边界。
```

当前数据应该这样归类：

```text
测试设计属于 B；
实际结果表现出 C；
所以当前问题是位置间一致性差，并且疑似由算法测量目标漂移造成。
```

### 10.2 三个 pos 均值该选哪个

没有外部真实标准值时，没有严格标准答案。

但有一个明确原则：

```text
不能按组内极差最小来选。
```

因为组内极差小只说明稳定，不能说明准确。

当前如果只是临时分析，可以说：

```text
pos1 是当前标定位置，最接近标注参考；
pos2/pos3 相对 pos1 系统偏短；
所以当前算法链路里 pos1 最有参考依据。
```

但正式作为零件侧边长，必须有下面至少一种依据：

1. 外部标准长度证明某个 `pos` 准确。
2. 产线明确只固定用某个 `pos` 测量。
3. 每个 `pos` 独立标定后，三者都能接近真实值。
4. 算法改为检测真实物理端面，而不是高亮反光边界。

在这些条件满足前，不能把三个 `pos` 的某一个均值直接宣称为真实侧边长。

## 11. 一句话总结

```text
这批数据的意图是同一个零件换 3 个位置测同一个侧边长；
当前每个位置内部重复性很好，但三个位置之间均值差明显；
所以现在不是“选哪个均值”的问题，而是“跨位置一致性没有被证明”的问题。
临时看 pos1 最接近当前标定参考，但正式长度必须由外部标准值或明确固定工位来决定，不能靠最小极差选择。
```
