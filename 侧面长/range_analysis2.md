# 正确做法：换标定图还是重做真实边界测量

本文只回答方案和原理，不优化代码。

## 1. 先给结论

当前不建议简单换标定图。

更准确的结论是：

```text
如果产线最终只固定一个位置测量，可以基于那个固定位置重新标定。
如果验收要求同一个零件放到 3 个位置都能测同一个侧边长，就必须解决位置间一致性差的问题。
```

而当前位置间一致性差的主要风险不是“标定图选错一张”这么简单，而是：

```text
算法当前测的是高光/反光条的边界，
不是产品真实物理端面或真实外轮廓边界。
```

所以正确方向应该是：

1. 高光区域可以继续用作粗定位。
2. 最终长度端点不能来自高光条左右边缘。
3. 最终长度应该来自产品真实物理边界：外轮廓、左右端面、端面轮廓线、或经验证稳定对应真实端面的几何特征。
4. 还要增加跨位置一致性检查，而不是只看每个位置内部的静态重复性。

一句话：

```text
换标定图只能移动偏差；
检测真实物理边界才可能从根上解决三个 pos 均值不一致的问题。
```

## 2. 换标定图能解决什么，不能解决什么

### 2.1 换成 pos2 或 pos3 标定图

如果把标定图换成 `pos2` 或 `pos3`，结果很可能变成：

```text
被标定的那个 pos 更接近参考值；
其他 pos 又产生新的偏差。
```

例如现在是 `pos1` 被标定锚定，所以 `s1 pos1` 非常接近参考值。如果换成 `pos3` 标定，`pos3` 可能被拉回参考值，但 `pos1/pos2` 不一定还对。

这叫：

```text
把系统偏差转移到另一个位置。
```

它不能证明算法真的测到了同一条物理边。

### 2.2 每个 pos 单独标定

每个位置都做一套标定，理论上可以减少位置间系统偏差。

例如：

```text
pos1 用 pos1 标定参数
pos2 用 pos2 标定参数
pos3 用 pos3 标定参数
```

这比只用一张 `pos1` 标定图套全部位置更合理。

但它有一个前提：

```text
每个位置下，算法测到的特征必须和真实物理边界有稳定关系。
```

如果算法测的是高光条，而高光条会随着光照、姿态、表面状态变化，那么每个位置单独标定也可能只是“拟合当前样本”，不是稳定测量真实长度。

所以单独标定只能作为辅助，不是根本解法。

### 2.3 什么时候换标定图是合理的

只有在下面这种情况下，换标定图才是合理方案：

```text
产线永远只在一个固定位置测量；
验收也只要求这个固定位置；
标定图就应该来自这个固定位置；
其他 pos 不参与最终输出。
```

如果客户要求：

```text
同一个零件放到 3 个位置都要输出一致的侧边长
```

那就不能靠换一张标定图解决。

## 3. 当前代码的问题本质

你列的这些缺陷判断是对的：

```text
没有检测零件外轮廓
没有检测左右端面
没有判断反光条是否覆盖了完整零件长度
没有区分真实材料边界和局部高光
```

当前算法的核心逻辑大致是：

1. 找图像中最明显的横向高亮长条。
2. 用这个高亮长条的左右边缘作为长度端点依据。
3. 用 `s1_label` 的人工线和高亮边缘之间的偏移做一次标定。
4. 把这个偏移套到所有位置。

这个思路的问题是：

```text
高光边界不是产品边界。
```

高光由光源、相机、角度、材料表面、曲率共同决定。零件只要换位置或转动一点，高光位置和长度就可能变化。

所以当前结果会出现：

```text
同一 pos 内部很稳定；
不同 pos 的均值明显不同。
```

这说明算法稳定地找到了某个视觉特征，但这个视觉特征未必是同一个物理边界。

## 4. 为什么不能继续只找高光区域

高光区域有价值，但只能作为粗定位或辅助信息。

可以把高光用于：

```text
定位零件大概在哪
估计零件轴线方向
缩小 ROI
判断图像是否曝光足够
判断拍摄是否异常
```

但不建议把高光条左右端直接当成产品长度端点。

原因是：

```text
真实边界是材料和背景、端面和侧面的几何分界；
高光边界是亮度分布分界。
```

这两个东西可能接近，但不是同一个东西。

在某些位置，高光刚好覆盖了完整长度，于是看起来对；换个位置，高光变短、变长或边缘缺失，算法仍然会稳定输出，但长度已经偏了。

## 5. 正确算法应该找什么

如果目标是测“零件侧边长”，最终要找的应该是下面这些之一：

1. 产品左右真实端面。
2. 产品外轮廓和轴线的交点。
3. 端面轮廓线拟合后的左右边界。
4. 与真实端面有稳定几何关系的结构边。
5. 经计量验证可以代表真实长度的边界特征。

也就是说，最终长度应该是：

```text
真实左端边界 到 真实右端边界
```

而不是：

```text
高光条左边界 到 高光条右边界
```

## 6. 可以用哪些算法寻找真实物理边界

下面按工程可落地程度列出几类方法。

### 6.1 外轮廓分割 + 轮廓拟合

思路：

1. 先把产品从背景中分割出来。
2. 得到产品整体 mask。
3. 从 mask 中提取产品外轮廓。
4. 拟合零件主轴。
5. 沿主轴方向找左右最外侧边界。
6. 用拟合结果计算长度。

可用方法：

```text
阈值分割
自适应阈值
背景差分
形态学开闭运算
连通域筛选
轮廓提取 findContours
最小外接矩形 minAreaRect
椭圆/圆弧/直线拟合
RANSAC 直线拟合
```

适用条件：

```text
产品和背景有稳定灰度差；
产品外轮廓在图像里清楚；
遮挡不严重。
```

优点：

```text
直接测产品整体几何，不依赖局部高光。
```

风险：

```text
如果产品很黑、背景也黑，外轮廓对比不够，单靠普通阈值会不稳定。
```

### 6.2 端面边缘检测 + 直线/圆弧拟合

如果左右端面在图像中能看到明显边缘，可以专门找端面。

思路：

1. 用高光或粗轮廓确定零件 ROI。
2. 在左端和右端附近分别取窄 ROI。
3. 对 ROI 做 Sobel/Canny 边缘检测。
4. 筛选接近垂直方向或端面方向的边缘点。
5. 用 RANSAC 拟合左端面线、右端面线。
6. 计算两条端面线在零件轴线上的距离。

可用方法：

```text
Sobel 梯度
Canny 边缘
Hough 直线
RANSAC 拟合
最小二乘直线拟合
亚像素边缘定位
```

如果零件是圆柱，端面在图像里可能不是一条直线，而是圆弧或椭圆边界。这时可以：

```text
拟合端面圆弧/椭圆
取轴线方向上的左右切点
计算两个切点之间的距离
```

适用条件：

```text
左右端面边缘可见；
端面边缘和背景/侧面有足够对比；
端面没有被夹具严重遮挡。
```

优点：

```text
直接测端面，比测高光可靠。
```

风险：

```text
如果端面边缘被反光、阴影或夹具遮挡，需要更好的打光或多视角。
```

### 6.3 沿零件轴线做一维边缘定位

如果产品主轴能稳定找到，可以沿轴线测长度。

思路：

1. 先估计零件轴线。
2. 沿轴线附近取一条或多条窄条带。
3. 对每条条带做灰度投影或梯度分析。
4. 找材料和背景之间的边界，而不是高光条边界。
5. 多条剖面结果做中位数或 RANSAC 融合。

可用方法：

```text
灰度剖面 profile
一阶梯度峰值
二阶导数/零交叉
亚像素插值
多剖面投票
RANSAC 剔除异常剖面
```

和当前算法的区别是：

```text
当前 profile 是在高光条区域找亮度突变；
正确 profile 应该跨过产品端面边界，找材料边界。
```

适用条件：

```text
端面附近灰度变化稳定；
轴线估计准确；
材料边界比高光边界更可重复。
```

### 6.4 模板匹配 / 形状匹配

如果产品形状固定，可以用标准模板找整个产品姿态和端点。

思路：

1. 建立标准边缘模板或 CAD 轮廓模板。
2. 在待测图中匹配产品整体位置、角度、尺度。
3. 根据匹配结果映射出左右端面位置。
4. 在端面附近做局部精定位。

可用方法：

```text
模板匹配
边缘模板匹配
Chamfer matching
形状上下文
特征点匹配
CAD 轮廓对齐
```

适用条件：

```text
产品形状固定；
夹具和背景变化不大；
整体轮廓可见。
```

优点：

```text
对局部高光不那么敏感；
能利用整体几何约束。
```

风险：

```text
如果轮廓被遮挡或形状变化大，模板匹配会失效。
```

### 6.5 监督式分割模型

如果传统方法难以稳定分割产品，可以训练一个分割模型。

思路：

1. 标注多张图中的产品 mask、左右端面或关键点。
2. 训练语义分割或关键点检测模型。
3. 模型输出产品区域、端面区域或端点。
4. 再用几何拟合做亚像素级测量。

可用方法：

```text
U-Net 类分割
Mask R-CNN 类实例分割
关键点检测
边缘分割网络
传统算法 + 深度学习融合
```

适用条件：

```text
传统视觉在反光、低对比、复杂背景下不稳定；
有足够代表性的标注数据；
可以接受训练和维护成本。
```

注意：

```text
深度学习可以帮助找到产品区域，
但最终高精度尺寸仍建议用几何拟合和标定计算。
```

## 7. 算法之外，打光可能更关键

对工业测量来说，很多时候不是先调算法，而是先让真实边界在图像里清楚。

如果当前图里产品是黑色圆柱，背景也暗，而且反光很强，那么算法天然会被高光吸引。

可以考虑的成像改进：

```text
背光：让产品形成清晰轮廓剪影
漫反射光：降低局部高光
偏振片：抑制金属/光滑表面反光
暗场光：突出边缘或划痕
同轴光：让平面端面更稳定显现
固定夹具：减少姿态变化
相机标定：消除畸变和位置比例差异
```

如果能用背光把产品外轮廓变成清晰剪影，那么测长度会简单很多：

```text
分割产品轮廓 -> 找左右端点 -> 拟合 -> 换算 mm
```

这通常比在强反光灰度图里硬找边界可靠。

## 8. 这样能解决当前问题吗

可以解决或显著缓解，但有前提。

### 8.1 能解决的情况

如果真实产品边界在图像里可见，只是当前代码没去找，那么改成“外轮廓/端面检测”后，理论上可以显著改善：

```text
pos1/pos2/pos3 均值不一致
反光条长度变化导致的测量偏差
局部高光误判
换位置后稳定偏短
```

因为最终测量目标从：

```text
亮度高光边界
```

变成：

```text
产品几何边界
```

这才符合“测侧边长”的物理定义。

### 8.2 不能保证解决的情况

如果真实边界在当前图像里不可见，例如：

```text
产品和背景几乎同黑
端面被夹具遮挡
边界被强反光覆盖
图像曝光不足
三个 pos 的像素比例不同但没有标定
```

那么单靠算法不一定能稳定找到真实边界。这时必须改打光、夹具、相机标定或拍摄方式。

算法不能凭空恢复图像里没有的信息。

## 9. 推荐的正确工程路线

### 9.1 第一步：明确尺寸定义

先定义：

```text
侧边长到底是哪两个物理边界之间的距离？
左端面哪条线？
右端面哪条线？
是在产品轴线上量，还是某条侧边线上量？
是否包含倒角、圆角、端盖、夹持区？
```

如果尺寸定义不清楚，算法会变成“看起来像长度”的测量，而不是可验收的计量。

### 9.2 第二步：建立真实标准值

需要至少有：

```text
标准件真实长度
或人工精标的端面点
或卡尺/CMM 测量值
```

用它判断：

```text
哪个算法结果更接近真实值。
```

没有真实标准值，就无法证明 `pos1`、`pos2`、`pos3` 哪个均值是对的。

### 9.3 第三步：先用高光做粗定位

高光不是完全不能用。

建议定位流程是：

```text
高光/亮条 -> 粗 ROI
产品整体区域 -> 精 ROI
端面/外轮廓 -> 最终测量边界
```

也就是：

```text
高光负责“找到大概位置”，
真实边界负责“最终计量”。
```

### 9.4 第四步：检测真实边界

优先尝试传统几何方法：

```text
产品外轮廓分割
端面边缘检测
轴线拟合
左右端面拟合
亚像素边缘定位
RANSAC 剔除异常点
```

如果传统方法在当前图像下不稳定，再考虑：

```text
改打光
加背光
加偏振
训练分割模型
模板匹配
```

### 9.5 第五步：增加质量判断

最终算法不能只输出长度，还应该输出质量指标。

例如：

```text
是否找到产品外轮廓
左右端面边缘点数量
端面拟合残差
端面边缘覆盖率
高光条是否覆盖完整长度
高光长度和外轮廓长度差异
pos 间均值差
是否超过位置间一致性阈值
```

这样才能避免“稳定测错还显示 OK”。

## 10. 当前阶段最合理的选择

当前阶段不应该先纠结：

```text
换哪张标定图？
选哪个 pos 的均值？
```

更应该先做：

```text
确认三个 pos 是否确实要求测同一物理侧边长；
确认真实侧边长标准值；
确认图像中真实端面/外轮廓是否可见；
确认是否要固定单一工位，还是三位置都要通过；
决定最终测量边界必须来自产品真实几何边界，而不是高光条。
```

如果验收要求三位置都可用，那么正确方向是：

```text
解决位置间一致性差；
改为检测真实产品边界；
必要时每个位置独立标定；
增加跨位置一致性判定。
```

如果验收只要求固定位置测量，那么可以：

```text
选择固定位置；
用该位置标定；
只输出该位置尺寸；
把其他位置作为鲁棒性验证，不混入最终长度。
```

## 11. 一句话总结

```text
换标定图不是根本解法。
当前根本问题是算法把高光条当成了长度边界。
正确做法是让高光只做粗定位，最终用外轮廓、左右端面、轴线和几何拟合来找真实物理边界。
如果真实边界在图像里可见，这能解决或明显改善三位置均值不一致；
如果真实边界不可见，就必须先改打光、夹具或相机标定。
```
