# Feature Specification: 137 壳体虚拟下位机基线

**Feature Branch**: `001-virtual-line-simulation`

**Created**: 2026-07-18

**Status**: Ready for planning

**Input**: 建立 137 壳体自动检测线的虚拟下位机基线：中台可控制设备状态并获取状态、
工件、报警和生产事件；服务按 OP10 至 OP80、资源互斥和多工件流水线进行仿真，且未来替换
真实 PLC 适配器时中台接口不变。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 控制并确认设备状态 (Priority: P1)

中台操作员提交启动、暂停、继续、停止、复位或回原点请求，并能明确看到该请求是否被受理、
设备当前状态及状态变化完成情况。

**Why this priority**: 没有可预测的启停和状态反馈，中台无法安全地演示或联调任何后续生产行为。

**Independent Test**: 在初始空闲设备上依次执行合法与非法控制请求，验证每次都得到明确结果，
且设备只按已批准的状态转换图变化。

**Acceptance Scenarios**:

1. **Given** 设备处于 Idle，**When** 中台提交新的启动请求，**Then** 请求被受理且设备进入
   Running。
2. **Given** 设备处于 Running，**When** 中台提交暂停再继续请求，**Then** 设备依次进入
   Paused 和 Running，暂停期间不开始新的工序。
3. **Given** 设备处于 Running 或 Paused，**When** 中台提交停止请求，**Then** 设备进入
   Stopped、所有在制品被标记为 Aborted，且必须完成回原点后才能重新启动。
4. **Given** 命令在当前状态不允许，**When** 中台提交该命令，**Then** 系统拒绝请求、说明原因，
   且设备状态不改变。
5. **Given** 中台因网络重试而提交相同的命令标识和相同内容，**When** 系统再次收到该请求，
   **Then** 系统返回首次结果且不重复执行动作。

---

### User Story 2 - 观察并行工件流转 (Priority: P2)

中台能够查看多个工件在 OP10 至 OP80 各工序中的位置、状态和占用资源，从而验证产线不是
“一件完成后才允许下一件”的单件串行模型。

**Why this priority**: 现有节拍分析表明首件约 125 秒、稳定生产约每 65 秒放行一件；并行流转是
虚拟下位机的核心业务价值。

**Independent Test**: 让仿真连续接纳至少三件工件，检查在某一时刻至少两件位于不同工序，且
任一容量为一的资源从未同时被两件工件占用。

**Acceptance Scenarios**:

1. **Given** 设备处于 Running，**When** 仿真连续接纳工件，**Then** 每件工件依次经过
   OP10、OP20、OP30、OP40、OP50、OP60、OP70 和 OP80。
2. **Given** 两件工件竞争同一资源，**When** 前一件仍在使用该资源，**Then** 后一件等待，
   不会产生双重占用。
3. **Given** 首件已按节拍到达螺纹检测平台的投料条件，**When** 下一件被接纳，**Then**
   前段工序可与后段工序并行，而非等待首件完全结束。
4. **Given** 单件的工序完成或失败，**When** 中台查询该工件，**Then** 可看到 SN（如已识别）、
   当前/最后工序、工序结果、资源和最终状态。

---

### User Story 3 - 处理报警与异常结果 (Priority: P3)

中台能收到可关联到设备、工件和工序的报警，并能按照恢复条件决定复位或回原点，避免把报警
误解为已自动恢复。

**Why this priority**: 在真实设备尚未到位时，先把故障语义、恢复路径和追溯数据确定下来，可
降低日后 PLC/设备联调返工。

**Independent Test**: 在运行中触发一个工序超时和一个配置错误，验证报警含代码、级别、时间和
来源；验证设备不能在故障未恢复时被复位为可生产状态。

**Acceptance Scenarios**:

1. **Given** 工序超过其配置的等待或执行时间，**When** 超时被判定，**Then** 系统记录关联工件
   和工序的报警，并按报警规则停止或结束受影响的流转。
2. **Given** 存在活动报警，**When** 中台请求复位，**Then** 系统只在故障恢复条件满足时离开
   Alarm；需要回原点的情况仍保持 Stopped，直到回原点完成。
3. **Given** 发生报警、状态变化或工件变化，**When** 中台订阅生产事件，**Then** 它收到带有
   唯一事件标识、发生时间和关联上下文的事件。

---

### User Story 4 - 查询生产记录和结果 (Priority: P4)

中台可按工件标识查看已完成、失败或中止的虚拟生产记录，并看到汇总结果及其形成过程，供演示
和后续追溯功能扩展使用。

**Why this priority**: 工件状态、报警和结果只有与唯一工件关联才可用于中台联调；完整生产数据
持久化和报表仍属于后续阶段。

**Independent Test**: 完成一件正常工件、失败一件工件并停止一件在制工件，分别查询三条记录，
验证每条记录都保留不同的最终状态和时间线。

**Acceptance Scenarios**:

1. **Given** 一件工件完成全部必需工序，**When** 中台查询该工件，**Then** 可查看工序时间线、
   结果和 OK/NG/Unknown 汇总状态。
2. **Given** 工件因停止被终止，**When** 中台查询该工件，**Then** 最终状态为 Aborted，且不会被
   后续调度继续推进。

### Edge Cases

- 同一命令标识被不同请求内容重复使用时，拒绝冲突请求且保留首次请求的结果。
- SN 识别失败、重复 SN、螺纹结果缺失、迟到或错配时，必须保留原始上下文并按待确认的业务规则
  标记异常；不得擅自把工件判为 OK。
- 暂停期间已经开始的物理原子动作的安全停止语义尚未确认；基线仿真只允许在工序边界冻结推进。
- 配置缺失、工序顺序错误或资源容量非法时，禁止开始生产并报告配置报警。
- 软件显示的急停或安全状态仅用于展示/诊断；不能替代硬件安全回路。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 将设备状态维护为 Idle、Running、Paused、Stopped 或 Alarm，并仅允许
  《设备与工件状态机》定义的转换。
- **FR-002**: 系统 MUST 接收启动、暂停、继续、停止、复位和回原点命令，并对无效状态中的请求
  返回不改变设备状态的明确结果。
- **FR-003**: 系统 MUST 使用命令唯一标识实现幂等：相同标识和内容返回首次处理结果；相同标识
  但不同内容被拒绝。
- **FR-004**: 系统 MUST 在收到停止命令后阻止新的工序推进，并将所有在制品标记为 Aborted。
- **FR-005**: 系统 MUST 只在 Stopped 完成回原点后回到 Idle；在 Stopped 时不得接受启动。
- **FR-006**: 系统 MUST 让工件依次流转 OP10 至 OP80，并记录工件标识、SN（如已识别）、工序
  状态、开始/结束时间、占用资源和最终状态。
- **FR-007**: 系统 MUST 支持多件工件同时存在于不同工序，并以资源容量保证互斥；任何容量为一的
  资源不得同时分配给两件工件。
- **FR-008**: 系统 MUST 从可配置的工序、资源、时长和超时规则驱动仿真，不得将这些业务值仅隐藏
  在程序逻辑中。
- **FR-009**: 系统 MUST 根据已知节拍目标支持首件约 125 秒和稳定投料间隔约 65 秒的仿真；各工序
  的临时时长必须可辨识为临时值。
- **FR-010**: 系统 MUST 记录带报警代码、严重级别、来源、时间、关联工件和工序的活动报警，并仅在
  恢复条件满足后完成复位。
- **FR-011**: 系统 MUST 向中台提供当前设备状态、工件查询、命令受理结果和生产事件；事件必须有
  唯一标识和发生时间。
- **FR-012**: 系统 MUST 将 PLC、机器人、相机、内窥镜和螺纹检测设备的特定通信细节隔离，确保
  中台业务接口在仿真与真实设备接入时保持稳定。
- **FR-013**: 系统 MUST 不把虚拟急停、软件报警或服务状态作为人员安全功能，并在相关状态中保留
  硬件安全边界说明。
- **FR-014**: 系统 MUST 为工件的 OK/NG/Aborted/Unknown 汇总结果保留形成依据；52 项结果、螺纹
  结果字典和最终 OK/NG 规则未确认前，不得把未知结果默认为 OK。

### Key Entities *(include if feature involves data)*

- **设备**: 产线的当前业务状态、活动命令、连接/安全诊断状态和活动报警集合。
- **命令**: 由中台提交的唯一控制请求、接收时间、处理结果和关联操作者。
- **工件会话**: 一件待检或已检测工件的唯一流转记录，包含 SN、生命周期、工序执行、资源占用和
  汇总结果。
- **工序执行**: 某工件在 OP10 至 OP80 中一个工序的状态、时间、结果和异常关联。
- **资源**: 可被工序独占或限量使用的设备能力，包含容量与当前所有者。
- **报警**: 已分类的异常记录，包含严重级别、来源、发生/恢复时间及设备/工件关联。
- **生产事件**: 面向中台的状态、工件、报警或命令完成通知，包含唯一标识和发生时间。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 对状态转换图中每个合法和非法的控制请求，验收场景均可重复验证，且非法请求零次
  改变设备状态。
- **SC-002**: 连续仿真三件工件时，至少一次可同时观察到两件工件处于不同工序，并且容量为一的
  资源双重占用次数为零。
- **SC-003**: 同一命令的至少三次网络重试只产生一次状态转换，且每次返回同一处理结果。
- **SC-004**: 每个注入的工序超时、配置错误和命令冲突均能在中台侧看到一条带代码、时间和关联
  上下文的报警或错误结果。
- **SC-005**: 所有完成、失败或中止的工件都可通过其工件标识查询到最终状态和工序时间线。
- **SC-006**: 在配置的基线仿真中，首件总目标为约 125 秒，后续工件的接纳间隔目标为约 65 秒；
  结果报告明确说明这不是最终现场验收结论。

## Assumptions

- 本功能仅提供虚拟下位机业务基线；真实 PLC、机器人、相机、内窥镜和螺纹设备尚未接入。
- 中台是已存在或将存在的调用方；认证、授权、网络部署和事件重放策略将在中台联调前确认。
- 真实 SN 格式、入料触发来源、重复 SN 处置、52 项检测结果和螺纹结果字典均未冻结；基线保留
  可关联和异常追溯能力，但不伪造最终质量规则。
- 125 秒首件目标和约 65 秒稳定投料间隔来自现有节拍分析；单工序仿真时长仅为临时配置。
- 180 秒 CT 的计时边界在现有材料中存在冲突，不能作为本功能的最终验收判断。
- 停止后在制品为 Aborted、回原点后才能重新开始，是当前已确认的业务规则。
- 持久化数据库、图片/点云保存、报表、MES/ERP、用户权限和 Docker 交付不属于本基线功能范围。
