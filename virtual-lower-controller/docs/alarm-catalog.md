# 报警目录（仿真初版）

## 使用说明

以下代码用于虚拟下位机的开发、接口和测试。它们不是已确认的现场 PLC 报警码；真实 PLC 和各设备供应商的故障代码到位后，由适配器建立映射，保留本服务稳定的业务报警分类。

严重级别：`Info`（提示）、`Warning`（可继续但需关注）、`Error`（当前工件/工序失败）、`Critical`（停止或禁止运行）。

## 处理与审计规则

- 报警事件至少包含：`eventId`、报警码、严重级别、来源、原始设备码（如有）、发生时间、关联 SN/工序、确认者和恢复时间。
- `Reset` 仅确认软件侧的恢复请求；不应掩盖仍存在的硬件/安全故障。
- 报警确认、报警清除、故障修复和回原点是不同动作；现场流程与权限待确认。
- 真实 PLC 报警映射、中文操作指导、优先级和最终 `requiresHome` 必须在联调前冻结。
