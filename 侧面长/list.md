# 当前还需要明确的问题清单

本文汇总前面关于静态重复性、三位置均值差异、标定和真实边界检测的讨论，列出后续必须确认的信息。

## 1. 尺寸定义必须明确

首先要明确“侧边长”到底是什么。

需要确认：

```text
测的是哪两个物理边界之间的距离？
左端点是左端面、左外轮廓，还是高光起点？
右端点是右端面、右外轮廓，还是高光终点？
长度是在产品轴线上量，还是沿某条侧边线量？
是否包含倒角、圆角、端盖、夹持区域？
如果端面是圆弧/椭圆投影，取哪个点作为长度端点？
```

如果尺寸定义不清楚，算法输出的只是“某个视觉长度”，不是可验收的工业尺寸。

## 2. 是否有真实标准长度

这是最关键的问题之一。

需要确认：

```text
有没有标准件？
标准件真实长度是多少 mm？
标准件长度由什么设备测得？
卡尺、千分尺、CMM 还是客户图纸值？
标准值的不确定度是多少？
sample_1 和 sample_2 是否有各自真实长度？
```

没有真实标准长度，就无法判断：

```text
pos1、pos2、pos3 哪个均值更接近真实长度。
```

也无法证明当前算法是“测得准”，只能证明“测得稳”。

## 3. 像素到毫米的标定关系

当前代码只有像素单位，没有正式 `px -> mm` 换算。

需要确认：

```text
mm_per_px 是多少？
px_per_mm 是多少？
这个比例来自哪里？
是否用标定板或标准件计算？
三个 pos 是否共用同一个比例？
不同位置是否需要独立标定？
是否做过相机畸变校正？
```

公式是：

```text
mm_per_px = 标准真实长度(mm) / 图像测得长度(px)
```

如果没有这个比例，就不能正式判断 0.1mm、0.05mm、0.03mm 是否满足。

## 4. 三个 pos 的业务含义

需要确认 `pos1/pos2/pos3` 到底代表什么。

可能有两种完全不同的含义：

```text
情况 A：同一个零件、同一个尺寸，放到 3 个不同位置拍摄。
情况 B：同一个零件上的 3 个不同物理测量位置。
```

如果是情况 A，则三个位置的均值应该接近，需要检查位置间一致性。

如果是情况 B，则三个位置可以本来就不同，必须分别和各自标准值比较。

目前从目录结构和数据采集方式看，更像是：

```text
同一个零件在 3 个不同拍摄位置重复测同一个侧边长。
```

但这需要业务方确认。

## 5. 最终长度到底取哪个 pos

需要明确最终报表口径。

必须确认：

```text
最终长度是只取 pos1？
还是 pos1、pos2、pos3 都要输出？
还是三个位置合成一个最终值？
如果合成，公式是什么？
如果某个 pos 异常，是否允许剔除？
剔除规则是什么？
```

当前不建议按下面方式选：

```text
选组内极差最小的 pos。
三个 pos 简单平均。
选最接近当前标定值的 pos。
手动去掉偏差最大的 pos。
```

原因是这些方法没有物理依据，可能把“稳定测错”的结果当成真实长度。

## 6. 静态重复性的验收公式

验收标准里写了：

```text
静态重复性 0.1mm
静态重复性 0.05mm
静态重复性 0.03mm
```

但还要确认具体公式。

需要明确：

```text
静态重复性 = max - min？
静态重复性 = 标准差 std？
静态重复性 = 3σ？
静态重复性 = 6σ？
是否使用 20 张图计算？
是否每个 pos 单独计算？
是否把三个 pos 混合计算？
```

当前代码输出的是：

```text
length_range_px = max(length_px) - min(length_px)
length_std_px = std(length_px)
```

但到底使用哪个指标验收，需要客户或验收文档确认。

## 7. 52 个尺寸项的清单

验收标准写：

```text
检测尺寸数量：52
0.1mm 重复性尺寸数量：10
0.05mm 重复性尺寸数量：10
0.03mm 重复性尺寸数量：32
```

需要拿到完整尺寸项清单。

每个尺寸项至少需要：

```text
尺寸编号
尺寸名称
所在产品区域
测量定义
名义值
上下公差
重复性等级
是否需要三位置一致
是否需要良品/不良品都输出
```

当前代码只测一个 `length_px`，还不是完整 52 个尺寸项验收报表。

## 8. 检出率 100% 的定义

需要明确“检出率 100%”到底指什么。

可能有几种口径：

```text
每张图片都能输出结果？
每个零件都能输出 52 个尺寸项？
600 件良品/不良品每件都能输出结果？
输出结果即可，还是必须质量判定 OK？
低置信度结果算不算检出？
fallback 结果算不算检出？
```

这会影响报表统计方式。

## 9. 良品和不良品数据要求

验收写了 600 件良品、不良品。

需要确认：

```text
600 件总数是多少良品、多少不良品？
每件是否都拍 3 个 pos？
每个 pos 是否都拍 20 张？
良品和不良品是否都要做重复性？
不良品的尺寸是否可能超出公差？
不良品是否仍要求输出尺寸值？
```

如果不良品形态改变明显，算法还要能稳定输出尺寸，而不是只适配良品。

## 10. 标定图和标定策略

当前标定基于：

```text
s1_label.json
s1_label.bmp
```

并且 `s1_label.bmp` 与 `sample_1/pos_1` 的一张图片相同。

需要确认：

```text
标定图是否应该来自固定工位？
是否每个 pos 都需要独立标定图？
标定图是否使用标准件？
标定线是否由人工标注真实端面？
人工标注误差如何控制？
是否需要多张标定图取平均？
标定参数是否长期有效？
换批次、换相机、换镜头后是否重新标定？
```

如果三个 pos 都要测同一个长度，只用 `pos1` 标定套全部位置风险较高。

## 11. 真实边界在图像里是否可见

当前算法主要找高光，不找真实产品边界。

需要确认：

```text
产品左右端面在图像中是否清晰可见？
产品外轮廓和背景是否有足够对比？
端面是否被夹具遮挡？
端面边缘是否被高光覆盖？
每个 pos 的端面可见性是否一致？
```

如果真实边界不可见，算法再复杂也无法可靠测量真实长度。

## 12. 是否允许改打光或夹具

如果当前图像中高光太强、产品和背景太接近，需要确认是否可以改成像方案。

需要确认：

```text
是否可以使用背光？
是否可以加漫反射光？
是否可以加偏振片抑制反光？
是否可以固定产品姿态？
是否可以调整相机角度？
是否可以换背景材料，提高轮廓对比？
是否可以避免夹具遮挡端面？
```

如果能让产品外轮廓变清晰，算法会简单可靠很多。

## 13. 是否要检测产品外轮廓和左右端面

从当前问题看，这项应该明确为后续算法目标。

需要确认：

```text
最终测量是否必须基于产品真实外轮廓？
是否必须检测左右端面？
高光是否只能作为粗定位？
是否需要输出端面拟合质量？
是否需要判断高光条是否覆盖完整零件长度？
```

建议后续算法目标写成：

```text
高光用于粗定位，最终尺寸来自真实产品几何边界。
```

## 14. 位置间一致性的验收阈值

当前 `pos1/pos2/pos3` 均值差明显。

需要确认：

```text
如果三个 pos 是同一尺寸，允许的位置间均值差是多少？
0.03mm？
0.05mm？
0.1mm？
还是另有再现性指标？
```

建议新增指标：

```text
position_mean_range_mm = (max(pos_mean) - min(pos_mean)) * mm_per_px
```

这个指标和静态重复性不同，应该单独验收。

## 15. 准确度验收标准

静态重复性只说明稳定，不说明准确。

还需要确认准确度要求：

```text
算法均值和真实标准值允许相差多少？
每个尺寸项的公差是多少？
输出尺寸是否要判 OK/NG？
测量误差超过多少报警？
```

建议至少有：

```text
bias_mm = algorithm_mean_mm - true_mm
abs(bias_mm) <= accuracy_limit_mm
```

## 17. 异常判定规则

需要明确什么情况下结果不能算合格。

例如：

```text
没有检测到产品外轮廓
没有检测到左端面
没有检测到右端面
端面拟合残差太大
边缘点数量不足
高光条没有覆盖完整长度
边缘置信度低
跨位置差异超限
长度偏离标准值超限
```

不能只要算法输出一个数就算 OK。

## 18. 当前最需要马上确认的 8 件事

优先级最高的是下面 8 项：

1. `pos1/pos2/pos3` 是否确实是同一个零件、同一个侧边长的三次换位拍摄。
2. 侧边长的真实物理定义是什么。
3. 有没有标准件长度或外部计量值。
4. `mm_per_px` 是多少，是否每个 pos 独立。
5. 静态重复性到底用 `max-min`、`std`、`3σ` 还是 `6σ`。
6. 三个 pos 是否都必须参与最终验收。
7. 真实左右端面或外轮廓在图像里是否可见。
8. 是否允许改打光、夹具或相机标定。

这 8 件事没明确前，不建议继续争论“选哪个 pos 均值”，也不建议直接把当前高光测量结果当成最终侧边长。

## 19. 一句话总结

```text
现在最缺的不是代码参数，而是计量定义和验收口径：
真实长度是多少、1px 等于多少 mm、三个 pos 到底是不是同一尺寸、重复性公式是什么、最终长度取哪个位置、真实边界是否可见。
这些问题明确后，才能判断是固定工位标定、每个 pos 独立标定，还是必须重做外轮廓/端面测量算法。
```
